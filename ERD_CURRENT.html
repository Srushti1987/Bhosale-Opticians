Database Relationships:

  • USERS → ORDERS: One-to-Many (One user can place multiple orders)
  • ORDERS → ORDER_ITEMS: One-to-Many (One order can have multiple items)
  • PRODUCTS → ORDER_ITEMS: One-to-Many (One product can be in multiple orders)
  • USERS → FEEDBACK: One-to-Many (One user can give multiple feedback)
  • ORDERS → FEEDBACK: One-to-One (Each order can have one feedback)
  • PASSWORD_RESETS: Independent table for password reset tokens

Table Descriptions:

  • USERS: Stores customer and admin information with role-based access
  • PRODUCTS: Product catalog with eyeglasses and sunglasses details
  • ORDERS: Customer orders with status tracking and payment information
  • ORDER_ITEMS: Individual products within each order with quantity and price
  • FEEDBACK: Customer feedback and ratings for completed orders
  • PASSWORD_RESETS: Temporary tokens for password reset functionality
