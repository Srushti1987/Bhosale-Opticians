DATABASE STRUCTURE - 11 TABLES
================================================================================

1. USERS TABLE
   - Stores customer information
   - Primary Key: user_id
   - Fields: username, email, password, phone, address, city, state, pincode, status

2. ADMIN TABLE
   - Stores administrator accounts (separate from users)
   - Primary Key: admin_id
   - Fields: username, email, password, phone, role, status, last_login

3. PRODUCTS TABLE
   - Product catalog for eyeglasses and sunglasses
   - Primary Key: product_id
   - Fields: product_name, gender, category, brand, model, price, image, etc.

4. PRODUCT_IMAGES TABLE
   - Multiple images per product
   - Primary Key: id
   - Foreign Key: product_id → products

5. CART TABLE
   - Shopping cart storage
   - Primary Key: cart_id
   - Foreign Keys: user_id → users, product_id → products

6. ORDERS TABLE
   - Customer orders
   - Primary Key: id
   - Foreign Key: user_id → users
   - Fields: order_number, total_amount, status, payment_method, shipping_address

7. ORDER_ITEMS TABLE
   - Individual items in each order
   - Primary Key: id
   - Foreign Keys: order_id → orders, product_id → products

8. BILL TABLE
   - Invoice/bill generation
   - Primary Key: bill_id
   - Foreign Key: order_id → orders
   - Fields: bill_number, bill_date, total_amount, tax_amount, subtotal

9. FEEDBACK TABLE
   - Customer feedback and ratings
   - Primary Key: id
   - Foreign Keys: user_id → users, order_id → orders, product_id → products
   - Fields: rating (1-5), feedback_text, admin_response

10. STOCK TABLE
    - Inventory management
    - Primary Key: id
    - Foreign Keys: product_id → products, updated_by → admin
    - Fields: quantity, reorder_level

11. PASSWORD_RESETS TABLE
    - Password reset tokens
    - Primary Key: id
    - Fields: email, token, user_type, expires_at

================================================================================
KEY RELATIONSHIPS
================================================================================

1:N Relationships (One-to-Many):
- USER → ORDERS (One user can place many orders)
- USER → CART (One user can have many cart items)
- USER → FEEDBACK (One user can give many feedback)
- ADMIN → STOCK (One admin can update many stock records)
- ADMIN → FEEDBACK (One admin can respond to many feedback)
- PRODUCT → PRODUCT_IMAGES (One product can have many images)
- PRODUCT → ORDER_ITEMS (One product can be in many orders)
- ORDER → ORDER_ITEMS (One order contains many items)

1:1 Relationships (One-to-One):
- ORDER → BILL (One order generates one bill)
- ORDER → FEEDBACK (One order can have one feedback)
- PRODUCT → STOCK (One product has one stock record)

HOW TO CREATE CHEN NOTATION ERD
================================================================================

Since HTML cannot perfectly create Chen notation ERD with ovals and diamonds,
use one of these professional tools:

OPTION 1: Draw.io (FREE - RECOMMENDED)
1. Visit: https://app.diagrams.net/
2. Create New Diagram → Blank Diagram
3. Use shapes:
   - Rectangle for Entities (USER, ADMIN, PRODUCT, etc.)
   - Oval/Ellipse for Attributes (user_id, email, etc.)
   - Diamond for Relationships (Placing Order, View, etc.)
4. Add cardinality labels (1, M) on connecting lines
5. Underline primary key attributes
6. Export as PNG/PDF

OPTION 2: Lucidchart (FREE TRIAL)
1. Visit: https://www.lucidchart.com/
2. Sign up for free account
3. Create new document
4. Choose "Entity Relationship" template
5. Select "Chen Notation" style
6. Drag and drop shapes

OPTION 3: MySQL Workbench (AUTO-GENERATE)
1. Open MySQL Workbench
2. Database → Reverse Engineer
3. Select sunray_db database
4. Auto-generates ERD from actual database
5. Export as image
