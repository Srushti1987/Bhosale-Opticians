✅ CART NaN ERROR - FIXED!
================================

PROBLEM:
--------
The shopping cart was showing "NaN" (Not a Number) errors because:
- Old cart.php used JavaScript localStorage
- Price format "₹2,999.00" couldn't be parsed correctly
- JavaScript calculations failed

SOLUTION:
---------
Created a SESSION-BASED cart system (no localStorage, no JavaScript parsing):
- add-to-cart.php: Adds products to PHP SESSION
- cart-session.php: Displays cart from SESSION
- checkout-session.php: Checkout using SESSION data
- Prices stored as floats (numbers), not strings
