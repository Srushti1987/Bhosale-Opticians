FILE STRUCTURE:
---------------
Root Files:
- login.php (Login for both admin and user)
- register.php (User registration)
- forgot-password.php (Request password reset)
- reset-password.php (Reset password with token)
- logout.php (Logout functionality)

User Dashboard:
- user/dashboard.php (User dashboard with stats)
- user/profile.php (Update profile & change password)
- user/orders.php (View order history)

Admin Dashboard:
- admin/dashboard.php (Admin dashboard with statistics)
- admin/products.php (Manage products - Add/Edit/Delete)
- admin/orders.php (Manage orders)
- admin/users.php (Manage users)

HOW IT WORKS:
-------------

1. REGISTRATION:
   - Users register at register.php
   - Password is hashed using PHP password_hash()
   - Default role is 'user'
   - Admin accounts must be created manually in database

2. LOGIN:
   - Both admin and user login at login.php
   - System checks role and redirects:
     * Admin → admin/dashboard.php
     * User → user/dashboard.php

3. FORGOT PASSWORD:
   - User enters email at forgot-password.php
   - System generates unique token
   - Token expires in 1 hour
   - Reset link is displayed (in production, send via email)
   - User clicks link to reset-password.php?token=xxx
   - User enters new password
   - Token is deleted after successful reset

4. ADMIN FEATURES:
   - View statistics (products, orders, users, revenue)
   - Add new products with image URL
   - Edit existing products
   - Delete products
   - View all orders
   - Manage users
   - Update order status

5. USER FEATURES:
   - View personal dashboard
   - See order history
   - Update profile information
   - Change password
   - Shop products
   - Add to cart
   - Place orders

DATABASE TABLES:
----------------
1. users
   - id, name, email, password, role, created_at
   - role: 'user' or 'admin'

2. password_resets
   - id, email, token, created_at, expires_at
   - Stores temporary reset tokens

3. products
   - id, name, description, price, category, image_url, on_sale, stock

4. orders
   - id, user_id, total_amount, status, shipping_address, created_at

5. order_items
   - id, order_id, product_id, quantity, price
