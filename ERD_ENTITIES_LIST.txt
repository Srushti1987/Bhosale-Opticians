ALL 11 ENTITIES (TABLES)

1. USERS
   Primary Key: user_id (INT)
   Purpose: Stores customer/user account information
   Attributes:
   - user_id (PK)
   - username
   - email
   - password
   - address
   - phone
   - city
   - state
   - pincode
   - status
   - created_at
   - updated_at

2. ADMIN
   Primary Key: admin_id (INT)
   Purpose: Stores administrator account information (separate from users)
   Attributes:
   - admin_id (PK)
   - username
   - email
   - password
   - phone
   - role
   - status
   - last_login
   - created_at
   - updated_at

3. PRODUCTS
   Primary Key: product_id (INT)
   Purpose: Stores eyeglasses and sunglasses product catalog
   Attributes:
   - product_id (PK)
   - product_name
   - gender
   - description
   - image
   - category
   - discount_percent
   - brand
   - frame_material
   - model
   - badge
   - lens_type
   - created_at
   - updated_at

4. PRODUCT_IMAGES
   Primary Key: id (INT)
   Foreign Key: product_id → PRODUCTS
   Purpose: Stores multiple images for each product
   Attributes:
   - id (PK)
   - product_id (FK)
   - image_url
   - alt_text
   - is_primary
   - sort_order
   - created_at

5. CART
   Primary Key: cart_id (INT)
   Foreign Keys: user_id → USERS, product_id → PRODUCTS
   Purpose: Stores shopping cart items for logged-in users
   Attributes:
   - cart_id (PK)
   - user_id (FK)
   - product_id (FK)
   - quantity
   - added_at
   - updated_at

6. ORDERS
   Primary Key: id (INT)
   Foreign Key: user_id → USERS
   Purpose: Stores customer order information
   Attributes:
   - id (PK)
   - order_number
   - user_id (FK)
   - total_amount
   - tax_amount
   - discount_amount
   - final_amount
   - status
   - payment_method
   - payment_status
   - payment_id
   - billing_address
   - shipping_address
   - notes
   - created_at
   - updated_at

7. ORDER_ITEMS
   Primary Key: id (INT)
   Foreign Keys: order_id → ORDERS, product_id → PRODUCTS
   Purpose: Stores individual line items within each order
   Attributes:
   - id (PK)
   - order_id (FK)
   - product_id (FK)
   - product_name
   - quantity
   - unit_price
   - total_price
   - discount_percent
   - discount_amount
   - created_at

8. BILL
   Primary Key: bill_id (INT)
   Foreign Key: order_id → ORDERS
   Purpose: Stores invoice/bill information for each order
   Attributes:
   - bill_id (PK)
   - order_id (FK)
   - bill_number
   - bill_date
   - subtotal
   - tax_amount
   - total_amount
   - created_at

9. FEEDBACK
   Primary Key: id (INT)
   Foreign Keys: user_id → USERS, order_id → ORDERS, product_id → PRODUCTS, responded_by → ADMIN
   Purpose: Stores customer feedback and ratings with admin responses
   Attributes:
   - id (PK)
   - user_id (FK)
   - order_id (FK)
   - product_id (FK)
   - rating
   - feedback_text
   - feedback_type
   - status
   - admin_response
   - responded_by (FK)
   - responded_at
   - created_at

10. STOCK
    Primary Key: id (INT)
    Foreign Keys: product_id → PRODUCTS, updated_by → ADMIN
    Purpose: Stores inventory/stock information for products
    Attributes:
    - id (PK)
    - product_id (FK)
    - quantity
    - reorder_level
    - updated_by (FK)
    - last_updated

11. PASSWORD_RESETS
    Primary Key: id (INT)
    Purpose: Stores password reset tokens for users and admins
    Attributes:
    - id (PK)
    - email
    - token
    - user_type
    - created_at
    - expires_at
    - used_at

--------------------------------
1. ORDERS → BILL (One order generates one bill)
2. ORDERS → FEEDBACK (One order has one feedback)
3. PRODUCTS → STOCK (One product has one stock record)

================================================================================
ENTITY PURPOSE & BUSINESS LOGIC
================================================================================

1. USERS
   - Stores customer registration data
   - Used for login authentication
   - Links to orders, cart, and feedback

2. ADMIN
   - Separate admin accounts (not mixed with users)
   - Role-based access control
   - Tracks who updated stock and responded to feedback

3. PRODUCTS
   - Central product catalog
   - Contains all eyeglasses and sunglasses
   - Categories: Men, Women, Kids

4. PRODUCT_IMAGES
   - Multiple images per product
   - Primary image designation
   - Sort order for display

5. CART
   - Persistent shopping cart
   - Stores items before checkout
   - Quantity tracking

6. ORDERS
   - Master order record
   - Payment and shipping details
   - Order status tracking

7. ORDER_ITEMS
   - Individual products in order
   - Quantity and pricing details
   - Discount calculations

8. BILL
   - Invoice generation
   - Tax calculations (18% GST)
   - Subtotal and total amounts

9. FEEDBACK
   - Customer ratings (1-5 stars)
   - Product and order feedback
   - Admin response capability

10. STOCK
    - Inventory tracking
    - Reorder level alerts (≤10)
    - Admin update tracking

11. PASSWORD_RESETS
    - Secure password reset tokens
    - 1-hour expiry
    - Separate for users and admins
