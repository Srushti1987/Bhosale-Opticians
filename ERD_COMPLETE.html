Database Relationships:

  • USERS → CART: 1:N (One user can have multiple cart items)
  • USERS → ORDERS: 1:N (One user can place multiple orders)
  • USERS → FEEDBACK: 1:N (One user can give multiple feedback)
  • PRODUCTS → PRODUCT_IMAGES: 1:N (One product can have multiple images)
  • PRODUCTS → CART: 1:N (One product can be in multiple carts)
  • PRODUCTS → STOCK: 1:1 (Each product has one stock record)
  • PRODUCTS → ORDER_ITEMS: 1:N (One product can be in multiple orders)
  • PRODUCTS → FEEDBACK: 1:N (One product can have multiple feedback)
  • ORDERS → ORDER_ITEMS: 1:N (One order contains multiple items)
  • ORDERS → FEEDBACK: 1:1 (Each order can have one feedback)
  • ADMIN → FEEDBACK: 1:N (One admin can respond to multiple feedback)
  • ADMIN → STOCK: 1:N (One admin can update multiple stock records)

Table Descriptions:

  • USERS: Customer accounts with complete profile information including address details
  • ADMIN: Administrator accounts with role-based access control (separate from users)
  • PRODUCTS: Complete product catalog with eyeglasses and sunglasses specifications
  • PRODUCT_IMAGES: Multiple images per product with primary image designation
  • CART: Persistent shopping cart storage for logged-in users
  • ORDERS: Customer orders with comprehensive payment and shipping information
  • ORDER_ITEMS: Individual line items within each order with pricing details
  • FEEDBACK: Customer feedback and ratings with admin response capability
  • STOCK: Inventory management with reorder levels and admin tracking
  • PASSWORD_RESETS: Secure password reset tokens for both users and admins
